Rencontre client

C'est une étape cruciale ! Vous devez bien comprendre les attentes du client pour être capable de faire des choix de design et de structure qui répondront aux besoins de la compagnie en matière de site web.

L'écoute est de mise, mais n'hésitez pas à poser des questions pour éclaircir certains points.

  • Quelles sont les fonctionnalités désirées ? Quel est son budget ?
  • Quel est le délai de livraison ?
  • Quelle est sa clientèle ?
  • Un site est-il déjà en ligne ?
  • Que veut-il améliorier ou garder ?

En posant les bonnes questions, vous allez diminuer les changements de directions, les ajouts ou retraits qui pourraient avoir une incidence sur le calendrier de production et les coûts de production.

déterminer
les technologies
à utiliser

En fonction des besoins de votre client, vous allez devoir determiner quels types de technologies doivent être utilisées pour mener à bien le projet.

Pour le côté utilisateur (front-end), les essentiels seront HTML, CSS et JS. Au-delà des langages de base, quel framework serait le plus adapté pour le travail ? React ? Vue.js ? Angular ?

Le site sera-t-il statique ou dynamique ? Un API sera-t-il nécessaire et aurez-vous besoin d'une base de données, donc un des technologies back-end ?

Ces points auront une incidende sur les coûts de production et les besoins en main-d'oœvre qualifiée.
